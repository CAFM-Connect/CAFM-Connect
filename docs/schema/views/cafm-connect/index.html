How Is A CAFM-Connect File Structured?

Every file, that is based on the "CAFM-Connect" specification is a native IFC file. In IFC there are two different file formats: STEP and IfcXML. A CAFM-Connect file is a file in the format IfcXML.

  • Filename extension: *.ifcxml
  • IFC version: IFC4 (final)
  • IFC specification: http://www.buildingsmart-tech.org/ifc/IFC4/final/html]
  • IFC scheme file (xsd): http://www.buildingsmart-tech.org/ifcXML/IFC4/Add1/IFC4_ADD1.xsd

In case of conflicts between the specification explained on this page and the officical IFC specification the IFC specification wins.

In CAFM-Connect there are two type of files:

  • Catalogue file: This file contains the units, classifications and property sets of CAFM-Connect
  • Data file: This file contains the content of the catalogue file (embedded catalogue) and the data itself

The following figure explaines the structure of a CAFM-Connect file. It shows the code blocks contained in the files.
